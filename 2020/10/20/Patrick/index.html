初始化操作过程

在本地建立 blog 文件夹
进入 Git 页面

pwd （查询目录所在位置）
mkdir Blog （在目录下建立文件夹 Blog）（如果在以后的操作过程中出现问题无法解
决就把这个文件夹删掉直接重新来过就可以了）
———————————————————————————————————————
cd Blog/ （进入 Blog 文件夹）
pwd （检测是否进入 Blog 文件夹）
———————————————————————————————————————
hexo init （即：初始化 hexo）
ls -l （查看初始化获得的文件）
hexo s （启动 hexo）

将网站部署到 Github 或者Gitee上

首先申请一个 github 账号，然后再 github 中按照 自己的昵称+.github.io 建立一个仓库（repository）（例如：patrick.github.io)

在 Blog 目录下安装一个 hexo 部署插件

找到 Blog 文件中的 _config.yml 文件，使用编译器打开，建议使用 Sublime Text 打开

Sublime Text 的好处：

1.支持的语言种类多

2.视图简洁

3.拥有插件种类多

4.别理，就是好看

打开后找到URL修改成自己的GitHub生成的连接，在这里找到然后修改

找到这部分代码，（原始的是没有 repo，和 branch 的，而且 type 后面也没有东西）
对这段代码进行更改
type 一个名字，建议写成 git 方便识别
repo repository 仓库地址 将刚刚新建的 github 仓库地址复制到这里就可以了
branch 分支这里设置管理员 main

——————————————————————————————————————————————————————

更改以后进行保存
